# Anthropic Agentic AI Engineering

This course explores the engineering of modern AI-powered software systems using agentic architectures and enterprise AI workflows.

Learners will learn how to:
- Build autonomous AI agents
- Design multi-agent systems
- Integrate AI tools using MCP (Model Context Protocol)
- Develop RAG-based knowledge systems
- Engineer AI-assisted software workflows
- Deploy and orchestrate production AI systems
- Apply governance, security, and responsible AI practices

The course uses industry-inspired concepts and workflows associated with Anthropic, Claude, MCP, and modern enterprise agentic AI platforms.

# Module 1: Introduction to Agentic AI

## 1.1 What is Agentic AI
- Autonomous reasoning systems — Understand how AI systems independently plan and execute tasks.
- AI agents vs chatbots — Compare reactive conversational systems with goal-oriented agents.
- Enterprise AI agents — Explore how organizations use AI agents in business workflows.
- Agentic workflows — Learn how agents coordinate multi-step processes autonomously.

## 1.2 Foundations of Modern AI Models
- Model families and capabilities — Examine modern AI model ecosystems and capabilities.
- Long-context processing — Learn how AI systems manage large contextual inputs.
- Constitutional and responsible AI — Understand safety-oriented AI design principles.
- Safety-oriented architectures — Explore techniques for building trustworthy AI systems.

## 1.3 AI API Fundamentals
- Messages and interaction APIs — Learn how applications communicate with AI systems programmatically.
- System prompts — Define AI behavior and operational constraints using system instructions.
- Streaming responses — Implement real-time AI response streaming for interactive applications.
- Structured outputs — Generate predictable schema-based AI responses.

## 1.4 Enterprise AI Use Cases
- Research agents — Build AI systems capable of autonomous information gathering.
- Coding assistants — Explore AI-assisted software development workflows.
- Enterprise copilots — Design AI assistants for organizational productivity.
- Workflow orchestration — Coordinate business processes using AI agents.

# Module 2: Prompt Engineering and Context Design

## 2.1 Prompt Engineering Principles
- Clear instruction design — Write precise prompts that improve AI reliability.
- Structured prompting — Organize prompts using semantic and XML-style structures.
- Deliberate reasoning — Encourage step-by-step reasoning for complex tasks.
- Context structuring — Organize contextual information for optimal AI performance.

## 2.2 Structured Agent Prompts
- Planner prompts — Design prompts that enable task planning and decomposition.
- Tool-routing prompts — Guide AI systems in selecting appropriate tools dynamically.
- Reflection prompts — Enable agents to evaluate and improve their own outputs.
- Evaluator prompts — Build prompts that assess quality, correctness, and safety.

## 2.3 Tool Use and Function Calling
- Tool schemas — Define structured interfaces for AI tool interaction.
- Function calling — Connect AI agents to executable backend functions.
- External API orchestration — Integrate third-party APIs into AI workflows.
- Safe tool execution — Implement safeguards for secure AI operations.

## 2.4 Context Engineering
- Long-context optimization — Manage large context windows efficiently.
- Retrieval strategies — Improve AI accuracy through intelligent information retrieval.
- Compression — Reduce unnecessary context while preserving important information.
- Context layering — Combine multiple contextual sources effectively.
